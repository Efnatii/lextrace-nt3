# AUDIT_RUNBOOK.md

## 1) Подготовка
1. Установите зависимости:
```bash
npm install
```
2. Соберите расширение:
```bash
npm run build
```
3. (Опционально) прогон автотестов:
```bash
npm run test:unit
npm run test:e2e
```

## 2) Загрузка unpacked в Microsoft Edge
1. Откройте `edge://extensions`.
2. Включите `Developer mode`.
3. Нажмите `Load unpacked` и выберите папку `dist` в корне проекта.
4. Убедитесь, что расширение `Neuro Translate` активировано.

## 3) Ручная проверка UI

### 3.1 Вкладки popup
1. Откройте любую `http/https` страницу.
2. Нажмите иконку расширения.
3. Проверьте, что есть ровно 3 вкладки:
- `Статус`
- `Настройки`
- `События`

### 3.2 Статус + disabled-логика
1. До старта перевода проверьте:
- `Показать перевод` disabled
- `Diff` disabled
- `Отмена` disabled
- `Стереть всё` disabled (если нет накопленных данных)
2. Нажмите `Запуск перевода`.
3. Проверьте, что прогресс и `done/pending/failed` обновляются.
4. Нажмите счётчик `err: N` (если `N>0`) и убедитесь, что открылась вкладка `События` с фильтром ошибок.

### 3.3 Настройки -> Профиль
1. Перейдите во вкладку `Настройки`.
2. Проверьте, что JSON-область профиля вертикально растягивается.
3. Нажмите по параметру в списке полей профиля.
4. Проверьте, что открыт один editor-контрол с одним input (`combobox`-стиль через datalist):
- тот же input и для выбора enum,
- и для свободного ввода.
5. Измените значение, нажмите Enter.
6. Проверьте, что профиль перешёл в `*` (dirty), а селектор профиля показывает `*`.

### 3.4 Настройки -> Доступ и ключи
1. Переключите `BYOK` / `PROXY`.
2. Проверьте маскированный ввод ключа/токена.
3. Проверьте `show/hide` кнопкой и `autocomplete=off`.

### 3.5 События
1. Перейдите во вкладку `События`.
2. Убедитесь в автообновлении списка.
3. Проверьте фильтрацию по категории и режим `только ошибки`.
4. Убедитесь, что записи раскрываются через `<details>`.
5. Нажмите:
- `Скачать лог`
- `Копировать лог`

## 4) Debug page и единый лог
1. В popup нажмите `⚙` (открыть debug).
2. Убедитесь, что debug страница показывает тот же поток событий.
3. Проверьте, что фильтры/экспорт в debug работают аналогично popup.

## 5) Ручная проверка пайплайна (C1..C5)

### C1: Инкрементальная подстановка
1. Запустите перевод на длинной странице.
2. До финала проверьте, что первые переводы уже подставляются в DOM.

### C2: Переключение режимов отображения
1. После появления переводов переключите `Перевод` / `Оригинал` / `Diff`.
2. Проверьте корректность отображения и disabled-правил.

### C3: Hard cancel mid-flight
1. Запустите перевод на большой странице.
2. Во время стадии `context/batching/translating` нажмите `Отмена`.
3. Убедитесь: стадия `cancelled`, очередь очищена, новые переводы не продолжаются.

### C4: API error -> лог + фильтр
1. Включите mock force error (или вызовите ошибку API).
2. Запустите перевод.
3. Убедитесь: состояние `failed`, событие ошибки в `События`, переход через счётчик ошибок работает.

### C5: Перезапуск SW/восстановление
1. Во время активного пайплайна перезапустите extension service worker (или весь extension).
2. Убедитесь, что не остаётся «подвешенного running»:
- orphan-сессия корректно помечается/перезапускается,
- состояние приходит в согласованное.

## 6) Где смотреть ключевые логи
- Popup events tab: runtime log list
- Debug page: тот же источник логов
- Persistent event store: IndexedDB (`neuro_translate_db`, store `event_logs`)

ТЗ / ИДЕЯ: Neuro Translate (переформулировано для максимально точной реализации, без изменения замысла)

АРХИТЕКТУРА РАСШИРЕНИЯ
1) Платформа
- Microsoft Edge (Chromium), Manifest V3.
- Контексты: Popup UI + Service Worker + Offscreen document (для стабильности пайплайна) + Content Script.

2) UI/UX (Popup)
- Визуальный стиль: ч/б flat.
- Компактно: без лишних поясняющих лейблов.
- Пояснения только через tooltip.
- Максимум иконок вместо текста, особенно на кнопках.
- Основная навигация: вкладки.

Вкладка 1 — «Статус»
- Progress bar + сводка выполнения перевода (сколько сделано/в работе).
- Счётчик ошибок: активный элемент. Клик → вкладка «События» + включён фильтр «ошибки».
- Кнопки-иконки:
  1) Запуск перевода.
  2) Жёсткая отмена: отмена на уровне API всех процессов, включая перевод.
  3) Стереть всё: удалить всю накопленную информацию.
  4) Переключение вида (3 кнопки): Показать перевод / Показать оригинал / Показать изменения (diff).
- Правила disabled/enabled (строго):
  - Нет переведённых блоков → «Показать перевод» и «Diff» disabled.
  - Процесс перевода не идёт → «Отмена» disabled.
  - Накопленной информации нет → «Стереть всё» disabled.

Вкладка 2 — «Настройки» (самая важная)
Содержит два блока.

Блок A: «Профиль»
- Возможность сохранять свой профиль.
- Просмотрщик JSON:
  - область можно вертикально расширять;
  - редактирование значений кликом по параметру;
  - выбор значения через реальный выпадающий список;
  - если поле допускает свободный ввод — ввод текста выполняется прямо внутри этого же выпадающего списка (без отдельного “всплывающего” инпута).
- Выбор активного профиля:
  - выпадающий список со всеми сохранёнными профилями;
  - параметры, на которые влияет профиль, подсвечиваются;
  - при изменении параметров профиль автоматически переходит в режим “*”.

Блок B: «Доступ и ключи»
- Внутри вкладки/подвкладки: BYOK и PROXY.

Вкладка 3 — «События» (критически важная)
- Авто-обновляемый лог абсолютно всех событий в максимально подробном виде.
- Записи лога — со сворачиваемыми секциями для компактности.
- Фильтрация по категориям событий.
- Действия:
  - скачать лог как файл;
  - копировать лог кнопкой.
- Во время отладки к этому же логу должен иметь доступ отладчик.

ПОДРОБНОСТИ РЕАЛИЗАЦИИ: ПАЙПЛАЙН
1) Запуск
- Нажатие «Перевод» запускает перевод активной страницы, на которой открыт popup.

2) Этап 1 — Аналитическое сканирование
- Найти все текстовые элементы.
- Разбить по категориям.
- Создать привязки/идентификаторы, чтобы подстановка не ломала DOM.
- Упорядочить элементы так, чтобы не терять/не ломать контекст при переводе.

3) Этап 2 — Глобальный контекст
- Все элементы (в правильном порядке) передаются ИИ контекста.
- Промпт максимально подробный.
- Результат: максимально подробный глобальный контекст, целевой объём не менее 15000 токенов.

4) Этап 3 — Батчинг
- Текстовые элементы объединяются в батчи (минимизация числа запросов к ИИ).

5) Этап 4 — Перевод батчей + инкрементальная подстановка
- Как только переведённый батч готов — он сразу подставляется в веб-страницу.
- ИИ перевода:
  - имеет максимально подробный системный промпт (как использовать глобальный контекст и уже переведённые батчи);
  - перевод идёт по списку батчей;
  - на каждом шаге передаются:
    - глобальный контекст,
    - несколько предыдущих переведённых батчей (контекстное окно),
    - компакции всех остальных батчей вне окна (для понимания общего контекста).
- ИИ перевода обязан возвращать structured output.
- Обязателен контроль TPM/RPM (OpenAI API).

НАСТРОЙКИ (в JSON-профиле)
- Prompt caching (настройки).
- Кол-во токенов для глобального контекста.
- Настройки батчей текстовых элементов.
- Настройки контекстного окна батчей:
  - батчи, вышедшие за окно, автоматически подвергаются автокомпакции;
  - компакция учитывает предыдущие компакции.
- Настройки компакции.
- Настройки хранения:
  - что хранить;
  - лимиты хранения;
  - при достижении лимитов освобождение места под новую информацию.
- Выбор моделей ИИ:
  - можно выбрать несколько моделей из списка;
  - список моделей берётся из OpenAI API;
  - модели сортируются по суммарной цене за 1M токенов (input+output) и цена показывается.
- Приоритет выбора модели:
  - отдельно для ИИ контекста;
  - отдельно для ИИ перевода.

ОГРАНИЧЕНИЯ И ПРАВИЛА
- Опираться на MV3 API / Edge API / OpenAI API.
- Максимум ООП и модульности.
- Модульные тесты + автоправки кода с повторными прогонами.
- E2E тесты на максимально сложной и хаотичной вёрстке, с правками и повторным тестированием.
